# BitRep Roadmap

This roadmap outlines the planned evolution of the BitRep protocol, implementation, and ecosystem. Timelines are approximate and may shift based on contributor input and security review.

## Q1
- stabilize identity and attestation modules
- finalize protocol specification
- publish architecture and threat model
- complete initial test suite
- release minimal governance module

## Q2
- add additional attestation import adapters (GitHub, LinkedIn)
- improve Sybil‑resistance heuristics
- introduce graph diagnostics endpoint
- expand reputation scoring tests
- publish integration examples

## Q3
- implement optional encrypted attestation storage
- add ZK threshold proof prototypes
- refine governance UX and proposal metadata
- introduce event‑driven reputation recomputation

## Q4
- evaluate alternative cryptographic primitives
- publish formal security review
- add multi‑service federation support
- prepare v1.0 protocol freeze

## Long‑Term
- decentralized attestation registries
- cross‑platform reputation portability
- advanced ZK privacy layer
- community‑driven governance extensions
