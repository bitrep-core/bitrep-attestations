# Contributing to BitRep

BitRep is an open protocol and reference implementation for verifiable, portable reputation. Contributions are welcome across cryptography, backend engineering, graph algorithms, governance design, and privacy systems.

## How to Contribute
- **[open an issue](guide://action?prefill=Tell%20me%20more%20about%3A%20open%20an%20issue)** — report bugs, propose features, or start design discussions  
- **[submit a pull request](guide://action?prefill=Tell%20me%20more%20about%3A%20submit%20a%20pull%20request)** — small, focused changes are easiest to review  
- **[join a discussion](guide://action?prefill=Tell%20me%20more%20about%3A%20join%20a%20discussion)** — comment on open issues or propose architectural improvements  

## Development Workflow
- **[fork the repository](guide://action?prefill=Tell%20me%20more%20about%3A%20fork%20the%20repository)** and create a feature branch  
- **[write clear, minimal commits](guide://action?prefill=Tell%20me%20more%20about%3A%20write%20clear%2C%20minimal%20commits)** with descriptive messages  
- **[add or update tests](guide://action?prefill=Tell%20me%20more%20about%3A%20add%20or%20update%20tests)** for any functional changes  
- **[run the full test suite](guide://action?prefill=Tell%20me%20more%20about%3A%20run%20the%20full%20test%20suite)** before submitting a PR  
- **[ensure API compatibility](guide://action?prefill=Tell%20me%20more%20about%3A%20ensure%20API%20compatibility)** unless the change is explicitly breaking  

## Code Standards
- **[type‑annotate all functions](guide://action?prefill=Tell%20me%20more%20about%3A%20type%E2%80%91annotate%20all%20functions)**  
- **[follow FastAPI conventions](guide://action?prefill=Tell%20me%20more%20about%3A%20follow%20FastAPI%20conventions)** for endpoints and responses  
- **[document new modules](guide://action?prefill=Tell%20me%20more%20about%3A%20document%20new%20modules)** with concise docstrings  
- **[avoid unnecessary dependencies](guide://action?prefill=Tell%20me%20more%20about%3A%20avoid%20unnecessary%20dependencies)** to keep the core lightweight  

## Security‑Related Contributions
- **[do not open public issues for vulnerabilities](guide://action?prefill=Tell%20me%20more%20about%3A%20do%20not%20open%20public%20issues%20for%20vulnerabilities)**  
- **[email the maintainer privately](guide://action?prefill=Tell%20me%20more%20about%3A%20email%20the%20maintainer%20privately)** for any security concerns  
- **[include reproduction steps](guide://action?prefill=Tell%20me%20more%20about%3A%20include%20reproduction%20steps)** and affected components  
- **[avoid speculative claims](guide://action?prefill=Tell%20me%20more%20about%3A%20avoid%20speculative%20claims)** — stick to demonstrable issues  

## What Makes a Good PR
- **[small, self‑contained changes](guide://action?prefill=Tell%20me%20more%20about%3A%20small%2C%20self%E2%80%91contained%20changes)**  
- **[clear motivation](guide://action?prefill=Tell%20me%20more%20about%3A%20clear%20motivation)** in the PR description  
- **[tests that cover edge cases](guide://action?prefill=Tell%20me%20more%20about%3A%20tests%20that%20cover%20edge%20cases)**  
- **[alignment with project architecture](guide://action?prefill=Tell%20me%20more%20about%3A%20alignment%20with%20project%20architecture)**  
- **[no unrelated refactors](guide://action?prefill=Tell%20me%20more%20about%3A%20no%20unrelated%20refactors)** in functional PRs  

## Areas Where Help Is Most Needed
- **[cryptographic review](guide://action?prefill=Tell%20me%20more%20about%3A%20cryptographic%20review)** of identity and attestation layers  
- **[graph‑scoring improvements](guide://action?prefill=Tell%20me%20more%20about%3A%20graph%E2%80%91scoring%20improvements)** to the reputation algorithm  
- **[governance model refinement](guide://action?prefill=Tell%20me%20more%20about%3A%20governance%20model%20refinement)** for proposals and voting  
- **[privacy enhancements](guide://action?prefill=Tell%20me%20more%20about%3A%20privacy%20enhancements)** to the ZK framework  
- **[integration adapters](guide://action?prefill=Tell%20me%20more%20about%3A%20integration%20adapters)** for additional platforms  

## Community Expectations
- **[be direct and constructive](guide://action?prefill=Tell%20me%20more%20about%3A%20be%20direct%20and%20constructive)**  
- **[focus on clarity over cleverness](guide://action?prefill=Tell%20me%20more%20about%3A%20focus%20on%20clarity%20over%20cleverness)**  
- **[respect architectural boundaries](guide://action?prefill=Tell%20me%20more%20about%3A%20respect%20architectural%20boundaries)**  
- **[assume good faith](guide://action?prefill=Tell%20me%20more%20about%3A%20assume%20good%20faith)** in discussions
